## 第九章：构建、打包与分发策略 (Build & Distribution)

本章定义了项目的工程化收口环节，包括如何利用 `tsdown` 抹平跨平台二进制的依赖差异，确保 CLI 工具在用户机器上的 100% 安装成功率。

### 9.1 工程结构与构建流水线 (Workspace & Build Pipeline)

项目采用标准的 Monorepo 或极简的单库结构。构建过程高度依赖 `tsdown` 的极速内联特性与 SvelteKit 的静态导出能力。

1.  **构建顺序 (Build Sequence)：**
    - **Step 1 (UI 构建)：** 优先触发 WebUI 目录下的 SvelteKit 构建 (`vite build`)。配置使用 `@sveltejs/adapter-static`，所有产物将被输出至全局的 `dist/webui/` 目录下。
    - **Step 2 (核心构建)：** 触发根目录的 `tsdown`。`tsdown` 会将 Node.js Daemon 逻辑和 CLI 入口代码一并打包、极致压缩，并输出至 `dist/`，同时内联绝大部分常规的 npm 依赖。
2.  **CLI 可执行文件声明：**
    - 在最终分发的 `package.json` 的 `bin` 字段中声明 `"pnpm-pub": "./dist/cli.js"`。

### 9.2 Native Addon 的差异化分发策略 (Dependency Strategy)

针对包含了 `.node` 预编译二进制的底层依赖，采取“分类治理”策略，彻底屏蔽因 `binding.gyp` 导致的用户端 C++ 编译报错。

1.  **对 `opentray` 的处理 (External 声明)：**
    - 由于 `opentray` 的原生包结构成熟，不强制触发本地编译，在 `tsdown.config.ts` 中将其明确标记为 `external`。
    - 它将作为常规的 `dependencies` 留在 `package.json` 中，由用户的包管理器（npm/pnpm）正常下载安装。
2.  **对 `@github/keytar` 的“胖包”内联处理 (Fat Package Plugin)：**
    - `@github/keytar` 带有 `binding.gyp`，极易导致安装失败。我们采取拦截并内联的分发策略。
    - **编写 `tsdown` Copy 插件：** 在打包时，将 `@github/keytar` 的 JS 源码直接内联到我们的产物中，并扫描其 `prebuilds/` 目录。
    - **精准剥离二进制：** 仅提取我们需要的目标平台预编译文件（`win32-x64`, `win32-arm64`, `darwin-x64`, `darwin-arm64`），并保持 `@github/keytar` 的包内布局复制到产物目录：`dist/prebuilds/keytar/prebuilds/<platform>/keytar.node`。
    - **CommonJS 边界：** 复制出的 `dist/prebuilds/keytar/package.json` 必须声明 `"type": "commonjs"`，避免根包 `"type": "module"` 改写 Keytar JS shim 的运行语义。
3.  **运行时动态挂载 (Dynamic Require)：**
    - 在 Daemon 源码中，禁止使用静态导入，改为基于当前系统环境动态加载拷贝好的二进制：

    ```javascript
    // pnpm-pub/src/daemon/keychain.ts
    import path from "node:path";

    const platform = process.platform; // 'win32' | 'darwin'
    const arch = process.arch; // 'x64' | 'arm64'

    // 指向我们通过 tsdown 插件复制过来的 Keytar JS shim；
    // shim 会按 @github/keytar 自身规则加载旁边的 prebuilds/<platform>/keytar.node。
    const keytarShimPath = path.join(__dirname, `../prebuilds/keytar/lib/keytar.js`);

    // 动态挂载
    const keytar = require(keytarShimPath);
    export default keytar;
    ```
    - **最终效果：** 用户执行 `npm i -g pnpm-pub` 时，无需任何 Python 或 C++ 环境即可秒级安装。整个包体积控制在几 MB 内，实现 Win/Mac 双端开箱即用。
