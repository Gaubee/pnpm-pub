## 第四章：数据结构与状态管理 (Data & Storage Design)

本章定义了项目在磁盘、操作系统凭据库以及前端视图层的数据模型与状态流转机制。

### 4.1 本地配置文件设计 (Local Configuration)

工具的公开配置统一存放在用户主目录下的 `~/.pnpm-pub/profiles.json` 文件中。该文件**绝对不包含**任何密码、Token 或 2FA Secret。

**数据模型定义 (TypeScript Interface):**

```typescript
interface PnpmPubConfig {
  /**
   * 默认选中的发包身份，指向 profiles 数组中的某一个 username
   */
  default: string;

  /**
   * 多身份配置列表
   */
  profiles: Array<{
    /** 用户名，作为该 Profile 的唯一 ID，与 NPM 保持一致 */
    username: string;
    /** NPM Registry 地址，默认为 https://registry.npmjs.org/ */
    registry?: string;
    /** 头像的远程 URL 或本地缓存标识 (可选) */
    avatarUrl?: string;
    /** 扩展字段：自动生成的 GitHub Action 脚本中使用的分支名等偏好 (可选) */
    ciPreferences?: Record<string, any>;
  }>;
}
```

- **状态流转：** 当 CLI 执行 `pnpm-pub start` 而未指定 `--profile` 时，Daemon 会读取此文件的 `default` 字段以加载默认身份。

### 4.2 系统 Keychain 凭据映射 (OS Credential Mapping)

由于 `@github/keytar` 的 API 需要 `service` (服务名) 和 `account` (账户名) 两个维度，我们将按照以下规范映射凭据，与 `profiles.json` 强关联：

- **Service Name (服务名):** 统一固定为 `"pnpm-pub"`。
- **Account Name (账户名) 规范:**
  - Token 存储键：`${username}_npm_token`
  - TOTP 密钥存储键：`${username}_totp_secret`

_例如，对于 username 为 `john_doe` 的用户，系统凭据中将存在两条记录：`john_doe_npm_token` 和 `john_doe_totp_secret`。_

### 4.3 缓存与日志目录结构设计 (Cache & Logs)

为了保持运行时的高效，Daemon 会在 `~/.pnpm-pub/` 下建立内部的缓存与日志机制：

- `~/.pnpm-pub/.cache/avatars/`: 存放用户头像的本地缓存图（如 `john_doe.png`）。在断网或 NPM 接口较慢时，托盘图标 (`opentray`) 仍能从这里秒读头像并正常渲染。
- `~/.pnpm-pub/logs/`: 存放 Daemon 的后台运行日志和 IPC 通信错误日志，方便问题排查。

### 4.4 SvelteKit 视图架构与路由状态 (Frontend State & Routing)

WebUI 抛弃轻量级状态机，采用工程化的 `SvelteKit` 配合原生的跨文档视图过渡。

1.  **静态适配器 (Static Adapter)：**
    配置 `@sveltejs/adapter-static` 将 SvelteKit 编译为纯静态的 SPA (Single Page Application)。编译产物打包至 `dist/webui`，由 Daemon 中的本地 HTTP 服务器对外 Serve。
2.  **路由与过渡动画 (Routing & View Transitions)：**
    - 利用 SvelteKit 的 `+layout.svelte` 构建全局的 Sidebar（多账号列表侧边栏）和托盘窗口的拖拽头部 (Drag Handle)。
    - 各个具体视图（如 `/` 看板，`/publish-confirm` 确认弹窗，`/add-profile` 新增账号）分散在对应的 `+page.svelte` 中。
    - 在路由切换时，利用浏览器现代的 `View Transitions API`，实现无需额外 JS 动画库的丝滑原生级过场效果。
3.  **全局状态同步 (Store & WebSocket)：**
    - 在 SvelteKit 的顶层维护一个长连接的 `WebSocket Client` 实例（携带 WebToken）。
    - 维护全局 Svelte Store（如 `$pendingPublish`），当收到 Daemon 发来的拦截发包消息时，更新 Store 并自动触发路由跳转到 `/publish-confirm` 页面进行物理确认。
