# OpenTray 窗口术语

更新时间：2026-07-17

| 术语                       | 中文共识               | 约束                                                                                            |
| -------------------------- | ---------------------- | ----------------------------------------------------------------------------------------------- |
| frameless                  | 无框窗口               | 移除原生标题栏与控件，不隐含某一种 Win32 resize 实现。                                          |
| production topology        | 生产拓扑               | 普通 OpenTray 消费端默认使用的 HWND/DWM/几何路径。                                              |
| comparator topology        | 对照拓扑               | 源码实验用于复现原生探针行为的内部路径，不是公开 API。                                          |
| compatibility bridge       | 兼容桥                 | pnpm-pub 在单一宿主边界投影内部变量的临时方案，必须具备回退和删除条件。                         |
| full-client                | 全客户区               | WebView 客户区覆盖整个 HWND，四边无原生非客户区 inset。                                         |
| native resize frame        | 原生缩放框             | 由 `WS_THICKFRAME`、DWM 非客户区和 `DefWindowProc` 共同管理。                                   |
| top-only client projection | 顶部单边客户区投影     | 左右和底部保留系统 resize inset；顶部只保留 DWM 可见边框。                                      |
| soft resize                | 模拟缩放               | 页面边缘命中后交给宿主启动 resize；不是原生非客户区。                                           |
| windowControlsOverlay      | 窗口控件覆盖层         | 保留系统窗口控件并让页面进入标题栏安全区；与 frameless 不同。                                   |
| retained window session    | 保留式窗口会话         | `close()` 只隐藏窗口并保留 WebView 页面状态；`toVisible()` 恢复同一会话。                       |
| bootstrap show             | 首次装载显示           | `show()` 仅用于首次加载原生扩展和创建页面运行时，不作为后续恢复动作。                           |
| operational visibility     | 操作可见性             | 窗口既未隐藏/关闭，也未最小化；由 `isVisible()` 与 `visibleChange` 表达。                       |
| native auto-hide           | 原生自动隐藏           | 失焦后由窗口宿主隐藏保留会话；`autoHide:false` 或 `keepOnTop:true` 会抑制。                     |
| page-owned auto-close      | 页面持有的自动关闭     | 页面负责退出动画/保护流程，宿主必须设置 `autoHide:false`，完成后调用 `close()`。                |
| switcher projection        | 任务切换器投影         | `style.platform.windows.showInSwitchers:false` 表示托盘工具窗不进入 Windows 任务栏或 Alt+Tab。  |
| visibility fact source     | 可见性事实源           | 只接受原生查询/事件；不得由 `document.visibilityState`、blur 或私有布尔值推断。                 |
| awaitable tray action      | 可等待的托盘动作       | 菜单回调返回操作队列的 Promise；事件源可忽略，但适配器可等待完成，且拒绝必须在队列内部收敛。    |
| stable WebView2 profile    | 稳定 WebView2 配置目录 | 由 OpenTray native host 按包版本与调用方标签隔离，不能由消费端按可执行文件路径自行推导。        |
| pnpx runtime path          | pnpx 运行时路径        | 临时安装目录和源码/全局安装目录不同；发布验收必须覆盖该路径。                                   |
| broker disconnect          | broker 断开            | WebView2/native host 初始化或生命周期失败时的进程级信号，不能仅按 WebUI 消失处理。              |
| published artifact parity  | 发布产物一致性         | 本地 workspace 与 npm tarball 必须分别验证，尤其是 native broker、WebView2 profile 和依赖版本。 |
